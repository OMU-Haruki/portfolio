JavaScriptの基本構文と特徴

2025年7月8日 JavaScript プログラミング Web開発

JavaScriptとは

JavaScriptは、ウェブページに動きや機能を追加するためのプログラミング言語です。1995年に作られて以来、現在では世界中のウェブサイトで使われています。

元々はウェブブラウザの中だけで動く言語でしたが、今ではサーバーやスマートフォンアプリの開発にも使われています。

本記事では、JavaScriptの基本的な構文や特徴について解説します。

JavaScriptの特徴

1. すぐに実行できる

JavaScriptはインタープリター言語と呼ばれます。これは、コンパイル（翻訳）という作業をしなくても、すぐにブラウザで実行できることを意味します。コードを書いたらすぐに結果を見ることができるので、学習や開発がしやすいです。

2. 変数の型を自動で判断してくれる

JavaScriptは動的型付けという仕組みを使っています。これは、変数に入れる値の種類（数字、文字など）を事前に決めなくても、自動で判断してくれることです。

let message = "こんにちは";    // 文字列
let number = 100;             // 数値
let isTrue = true;            // 真偽値（trueまたはfalse）

3. オブジェクト指向プログラミングができる

JavaScriptはオブジェクト指向プログラミングに対応しています。これにより、現実世界のものをプログラムで表現しやすくなります。例えば、「人」や「車」などをオブジェクトとして作ることができます。

基本的な構文

変数の作り方

変数は、値を入れておく箱のようなものです。JavaScriptでは3つの方法で変数を作ることができます：

var name = "太郎";         // 古い書き方
let age = 20;              // 新しい書き方（値を変更できる）
const city = "東京";        // 定数（値を変更できない）

現在はletとconstを使うことが推奨されています。

データの種類

JavaScriptで扱えるデータの種類を紹介します：

  • 文字列（string）：文字や文章を表します
  • 数値（number）：数字を表します
  • 真偽値（boolean）：trueまたはfalseの値
  • 配列（array）：複数の値をまとめて管理します
  • オブジェクト（object）：関連する情報をまとめて管理します

関数の作り方

関数は、まとまった処理に名前をつけて、何度も使えるようにするものです：

// 基本的な関数の書き方
function greet(name) {
    return "こんにちは、" + name + "さん！";
}

// 関数を使う
let message = greet("田中");
console.log(message); // 「こんにちは、田中さん！」と表示

条件分岐

条件によって処理を変えたい時に使います：

let score = 85;

if (score >= 90) {
    console.log("優秀です！");
} else if (score >= 70) {
    console.log("良い成績です");
} else {
    console.log("もう少し頑張りましょう");
}

繰り返し処理

同じ処理を何度も行いたい時に使います：

// 1から5まで数える
for (let i = 1; i <= 5; i++) {
    console.log(i + "回目");
}

// 配列の中身を1つずつ処理する
let fruits = ["りんご", "みかん", "バナナ"];
for (let fruit of fruits) {
    console.log(fruit + "が好きです");
}

配列とオブジェクトの使い方

配列

配列は、複数の値を順番に並べて管理するものです：

// 配列の作成
let colors = ["赤", "青", "緑"];

// 配列の操作
colors.push("黄色");        // 最後に追加
colors.pop();               // 最後を削除
console.log(colors[0]);     // 最初の要素を取得（"赤"）
console.log(colors.length); // 配列の長さを取得

オブジェクト

オブジェクトは、関連する情報をまとめて管理するものです：

// オブジェクトの作成
let person = {
    name: "田中太郎",
    age: 25,
    city: "大阪",
    greet: function() {
        return "はじめまして、" + this.name + "です";
    }
};

// オブジェクトの使用
console.log(person.name);     // "田中太郎"
console.log(person.greet());  // "はじめまして、田中太郎です"

便利な新機能

テンプレートリテラル

文字列の中に変数を埋め込みやすくする機能です：

let name = "佐藤";
let age = 30;

// 従来の書き方
let message1 = "私の名前は" + name + "で、" + age + "歳です";

// 新しい書き方（バッククォートを使用）
let message2 = `私の名前は${name}で、${age}歳です`;

アロー関数

関数をより短く書ける方法です：

// 従来の書き方
function add(a, b) {
    return a + b;
}

// アロー関数の書き方
const add2 = (a, b) => {
    return a + b;
};

// さらに短い書き方
const add3 = (a, b) => a + b;

分割代入

配列やオブジェクトから値を取り出しやすくする機能です：

// 配列の分割代入
let numbers = [1, 2, 3];
let [first, second, third] = numbers;

// オブジェクトの分割代入
let user = {name: "山田", age: 28};
let {name, age} = user;

実際に使ってみる

実際にJavaScriptを使って、私のポートフォリオサイトに機能を追加しました。実装した3つの機能を例に、JavaScriptがどのように活用されているかを解説します。

これらの機能は、基本的な構文を組み合わせることで作られており、JavaScriptの実用性と柔軟性を示しています。

1. トップへ戻るボタン

長いページをスクロールした際に、ユーザーが簡単にページの最上部に戻れるようにする機能です。この機能では、DOM操作、イベントリスナー、条件分岐、Web APIなど、JavaScriptの基本的な概念がすべて使われています。

機能の概要

  • ページが300px以上スクロールされたときにボタンが表示される
  • ボタンをクリックするとスムーズにページトップまでスクロールする
  • ボタンは固定位置に表示され、常にアクセス可能
// DOMが読み込まれてから実行
document.addEventListener('DOMContentLoaded', function() {
    // ボタン要素を作成
    const backToTopButton = document.createElement('button');
    backToTopButton.id = 'back-to-top';
    backToTopButton.innerHTML = '↑';
    
    // ページにボタンを追加
    document.body.appendChild(backToTopButton);
    
    // スクロール位置を監視
    window.addEventListener('scroll', function() {
        const scrollTop = window.pageYOffset;
        
        // 条件分岐：300px以上スクロールしたらボタンを表示
        if (scrollTop > 300) {
            backToTopButton.classList.add('show');
        } else {
            backToTopButton.classList.remove('show');
        }
    });
    
    // ボタンクリック時のスムーススクロール
    backToTopButton.addEventListener('click', function() {
        window.scrollTo({
            top: 0,
            behavior: 'smooth'
        });
    });
});

コードの詳細解説

1. DOMContentLoadedイベント

document.addEventListener('DOMContentLoaded', function() {...})は、HTMLの読み込みが完了してからJavaScriptを実行するための仕組みです。これにより、まだ存在しない要素にアクセスしようとするエラーを防げます。

2. 要素の動的作成

document.createElement('button')でボタン要素をプログラム的に作成し、innerHTMLでボタンの内容（↑）を設定しています。これは「DOM操作」と呼ばれる重要な技術です。

3. スクロール監視

window.addEventListener('scroll', ...)でスクロールイベントを監視し、window.pageYOffsetで現在のスクロール位置を取得します。300px以上スクロールした場合のみボタンを表示することで、ユーザビリティを向上させています。

4. CSSクラスの操作

classList.add('show')とclassList.remove('show')でCSSクラスを動的に追加・削除し、ボタンの表示・非表示を制御しています。これはJavaScriptとCSSの連携の良い例です。

5. スムーススクロール

window.scrollTo({top: 0, behavior: 'smooth'})は、ページトップまでなめらかにスクロールするWeb APIです。behavior: 'smooth'により、瞬間移動ではなく滑らかな動きを実現しています。

使用している技術：

  • DOM操作：createElement（要素作成）、appendChild（要素追加）
  • イベントリスナー：addEventListener（イベント監視）
  • 条件分岐：if-else文（状態に応じた処理分岐）
  • オブジェクトのメソッド：classList.add/remove（CSSクラス操作）
  • Web API：window.scrollTo（スクロール制御）

2. 画像カルーセル機能

プロジェクトページで複数の画像を1つのスペースで切り替えて表示する機能です。この機能は配列操作、JSON処理、循環アルゴリズムなど、より高度な概念を含んでいます。

機能の概要

  • 複数の画像を1つの表示エリアで順番に表示
  • 前へ・次へボタンで画像を手動切り替え
  • ドット（インジケーター）で現在表示中の画像を視覚的に表示
  • 10秒間隔での自動切り替え（ホバー時は停止）
// カルーセルの初期化
carousels.forEach(function(carousel) {
    // JSON.parseでデータ属性から配列を取得
    const images = JSON.parse(carousel.dataset.images);
    let currentIndex = 0;
    
    // 次の画像を表示する関数
    function showNextImage() {
        // 配列のインデックスを循環させる
        currentIndex = (currentIndex + 1) % images.length;
        updateImage();
    }
    
    // 前の画像を表示する関数
    function showPrevImage() {
        currentIndex = (currentIndex - 1 + images.length) % images.length;
        updateImage();
    }
    
    // 画像とドットの表示を更新
    function updateImage() {
        carouselImage.src = images[currentIndex].src;
        carouselImage.alt = images[currentIndex].alt;
        
        // 全てのドットから active クラスを削除
        dots.forEach(dot => dot.classList.remove('active'));
        // 現在のドットに active クラスを追加
        dots[currentIndex].classList.add('active');
    }
});

コードの詳細解説

1. forEachによる反復処理

carousels.forEach(function(carousel) {...})は、ページ内の全てのカルーセル要素に対して同じ処理を適用する配列メソッドです。これにより、複数のカルーセルが存在しても、それぞれ独立して動作させることができます。

2. JSON.parseによるデータ解析

JSON.parse(carousel.dataset.images)は、HTML要素のdata属性に保存されたJSON文字列を、JavaScriptの配列オブジェクトに変換します。これにより、HTMLとJavaScriptの間でデータをやり取りできます。

3. 循環アルゴリズム

(currentIndex + 1) % images.lengthは剰余演算子（%）を使った循環処理です。最後の画像の次は最初の画像に戻り、最初の画像の前は最後の画像になるという無限ループを実現しています。

4. 関数の定義と活用

showNextImage()、showPrevImage()、updateImage()という3つの関数を定義し、それぞれが特定の責任を持つことで、コードの可読性と保守性を向上させています。

5. DOM要素の属性変更

carouselImage.srcとcarouselImage.altを変更することで、表示する画像を動的に切り替えています。また、ドットのアクティブ状態も同期して更新しています。

使用している技術：

  • 配列の操作：forEach（反復処理）、インデックス操作（要素アクセス）
  • JSON操作：JSON.parse（文字列から配列への変換）
  • 剰余演算子：%による循環処理（無限ループの実現）
  • 関数の定義：処理の分割と再利用性の向上
  • DOM要素の属性変更：src、altの動的更新

3. プロジェクト説明の展開・折りたたみ機能

長い説明文を初期状態では省略して表示し、「もっと表示する」ボタンで全文を表示する機能です。この機能では文字列操作、条件判定、動的要素生成など、実用的なWebアプリケーションで頻繁に使われる技術が組み合わされています。

機能の概要

  • 説明文の長さを自動判定（段落数または文字数）
  • 長い説明文を自動的に折りたたんで表示
  • 「もっと表示する」ボタンで全文表示に切り替え
  • 「表示を戻す」ボタンで再び折りたたみ、自動スクロール機能付き
// 全ての説明文要素を取得
const projectDescriptions = document.querySelectorAll('.project-description');

projectDescriptions.forEach(function(description) {
    const textContent = description.textContent.trim();
    const paragraphs = description.querySelectorAll('p');
    
    // 条件判定：段落が3つ以上または文字数が300文字を超える場合
    if (paragraphs.length >= 3 || textContent.length > 300) {
        // 初期状態で折りたたみ
        description.classList.add('collapsed');
        
        // ボタンを動的に作成
        const toggleButton = document.createElement('button');
        toggleButton.className = 'toggle-description';
        toggleButton.textContent = 'もっと表示する';
        
        // クリック時の切り替え処理
        toggleButton.addEventListener('click', function() {
            const isCollapsed = description.classList.contains('collapsed');
            
            if (isCollapsed) {
                // 展開処理
                description.classList.remove('collapsed');
                toggleButton.textContent = '表示を戻す';
            } else {
                // 折りたたみ処理
                description.classList.add('collapsed');
                toggleButton.textContent = 'もっと表示する';
                
                // スムーススクロール
                description.scrollIntoView({
                    behavior: 'smooth',
                    block: 'start'
                });
            }
        });
        
        // ボタンを要素の後に挿入
        description.parentNode.insertBefore(toggleButton, description.nextSibling);
    }
});

コードの詳細解説

1. 要素の一括取得

document.querySelectorAll('.project-description')は、指定したCSSセレクターに一致する全ての要素をNodeListとして取得します。これにより、ページ内の全てのプロジェクト説明文に同じ処理を適用できます。

2. 文字列と要素の解析

textContent.trim()で前後の空白を除去した文字列を取得し、lengthプロパティで文字数を確認します。同時にquerySelectorAll('p')で段落数も取得し、複数の条件で判定を行います。

3. 論理演算子による複合条件

paragraphs.length >= 3 || textContent.length > 300では、論理OR演算子（||）を使って「段落が3つ以上」または「文字数が300文字超」のいずれかが真の場合に処理を実行します。

4. 状態管理と切り替え処理

classList.contains('collapsed')で現在の状態を確認し、その結果に基づいて表示を切り替えます。これは状態管理の基本的なパターンです。

5. DOM構造の操作

parentNode.insertBefore(toggleButton, description.nextSibling)は、既存のDOM構造に新しい要素を挿入する方法です。nextSiblingにより、説明文の直後にボタンを配置しています。

6. ユーザビリティの向上

折りたたみ時のscrollIntoView({behavior: 'smooth', block: 'start'})により、ユーザーが現在見ている位置を見失わないよう、適切な位置にスムーズにスクロールします。

使用している技術：

  • 配列のメソッド：querySelectorAll（要素の一括取得）、forEach（反復処理）
  • 文字列操作：textContent（テキスト取得）、trim()（空白除去）、length（文字数取得）
  • 論理演算子：||（OR演算子）を使った複合条件判定
  • 真偽値の活用：classList.contains（状態確認）による状態管理
  • DOM操作：insertBefore（要素挿入）、parentNode（親要素参照）
  • Web API：scrollIntoView（要素へのスクロール）

JavaScriptの実用性

これらの実例からわかるように、JavaScriptは基本的な構文を組み合わせることで、多彩で実用的なWebサイト機能を実現できます。以下の要素が特に重要な役割を果たしています：

  • DOM操作：HTML要素の動的な取得、作成、変更
  • イベント処理：ユーザーの操作に対するリアルタイムな反応
  • 配列とオブジェクト：複雑なデータの効率的な管理
  • Web API：ブラウザが提供する高機能な機能の活用

これらの実例を通して、JavaScriptが単なるプログラミング言語以上の存在であることがわかります。変数、関数、条件分岐、繰り返し処理といった基本構文を組み合わせることで、ユーザーにとって魅力的で使いやすいWebサイトを構築することができます。

DOM操作によるページの動的な変更、イベント処理によるユーザーとのインタラクション、配列とオブジェクトによる効率的なデータ管理など、JavaScriptの多面性が現代のWeb開発において不可欠な理由を示しています。

まとめ

本記事では、JavaScriptの概要から特徴、基本的な構文、実用例について解説しました。

JavaScriptは、ウェブ開発において中核的な役割を果たすプログラミング言語です。その特徴である動的型付け、インタープリター実行、オブジェクト指向対応により、開発者は迅速かつ柔軟にWebアプリケーションを構築できます。
